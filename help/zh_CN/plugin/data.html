保存插件数据

很多插件需要管理员或用户输入一些信息以便下次会话时使用这些数据，此时就必须保存这些数据，以下是保存数据的 3 种方法：

  1. 用户设置：此方法可由开发者利用 opts.php 配置，配置后该文件后，管理后台的插件管理中，会自动生成用户设置表单，用户设置后，数据将会保存到插件目录的 opts.json 文件中。在插件运行时，这些数据将会自动读取到插件对象中。详情请查看 编写插件 和 插件对象 的相应说明。
  2. 生成静态缓存：此方法可由开发者利用 PHP 的 Filesystem 生成静态的 json 等文件来保存数据，必要的时候再读取数据。文件建议保存在插件目录中，为了简化插件路径的获取，系统内置了一个常量来定义常见目录，开发者可随时调用，该常量名称为 BG_PATH_PLUGIN，例如：$path = BG_PATH_PLUGIN . 'hello/';。另外，系统也内置了一个操作文件和目录的文件类，在需要时，随时可以实例化该类，该类名称为 CLASS_FILE，例如：$obj_file = new CLASS_FILE();，关于该类的具体说明请查看 文件系统类。
  3. 保存至数据库：此方法可利用系统内置的全局对象将数据直接保存至数据库，该对象名称为 $GLOBALS['obj_db']，开发者可随时利用该对象操作数据库，关于该对象的具体说明请查看 数据库对象。
